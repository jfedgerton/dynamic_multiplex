--- Bug Fix: Self-Loop Similarity for Undirected Networks ----
  FILES: multilayer_utils.py (Python), multilayer_utils.R (R)

  PROBLEM: Community self-loops were always assigned similarity = 1.0,
  regardless of whether the network was directed or undirected. Louvain
  and Leiden treat undirected edges as two directed half-edges internally
  (A[i,j] + A[j,i]), so the self-loop should count 2.0 for undirected.

  FIX: Added a `directed` parameter to `add_community_self_loops()`.
  Set self_sim = 1.0 for directed, 2.0 for undirected. Both the raw
  `similarity` and `weighted_similarity` fields now reflect this.

  EVIDENCE: The same logic appears in both Python and R implementations
  with identical semantics.

--- Bug Fix: directed= Not Passed to add_community_self_loops ----
  FILES: fit_multilayer_jaccard.py, fit_multilayer_overlap.py,
         fit_multilayer_weighted_jaccard.py, fit_multilayer_weighted_overlap.py
         (and R equivalents)

  PROBLEM: The four fit functions did not forward their `directed`
  argument to `add_community_self_loops()`, so self-loops always used
  the default (False/undirected) behavior even for directed networks.

  FIX: Added `directed=directed` to every call to
  `add_community_self_loops()` in all four Python and all four R fit
  functions.

--- Bug Fix: Duplicate Self-Loops for Multi-Link Layers ----
  FILES: multilayer_utils.py (Python), multilayer_utils.R (R)

  PROBLEM: The old code iterated over each row in `layer_links` and
  created self-loops per the "from" layer. If a layer appeared in
  multiple links, it would get duplicate self-loop entries.

  FIX: Replaced the per-row iteration with a two-pass approach:
  (1) compute the max layer_weight for each unique layer across all
  links, then (2) emit exactly one self-loop per (layer, community)
  pair using that max weight.

--- Bug Fix: Leiden Objective Function for Undirected Graphs ----
  FILES: multilayer_utils.py (Python), multilayer_utils.R (R)

  PROBLEM: Leiden used `ModularityVertexPartition` for undirected
  graphs, which can behave unexpectedly with weighted multiplex
  networks (resolution limit issues).

  FIX: Changed to `RBConfigurationVertexPartition` for undirected
  graphs (Reichardt-Bornholdt with configurable resolution). Directed
  graphs still use `CPMVertexPartition`. The R equivalent uses
  objective_function = "modularity" → unchanged (igraph's Leiden R
  binding maps this correctly to the RB variant).

--- Removed: _communities_to_membership helper ----
  FILES: multilayer_utils.py (Python)

  Removed unused `_communities_to_membership()` function. Confirmed via
  full codebase search that no file references it.

--- New: Benchmark Scripts ----
  FILES: scripts/benchmark_python.py, scripts/benchmark_r.R,
         scripts/run_benchmarks.sh

  Added benchmark harness comparing dynamic_multiplex against:
    1. Independent Louvain (no temporal coupling)
    2. Full-coupling Louvain (all layer pairs)
    3. Aggregated Louvain (collapse layers)
    4. Spectral SBM (adjacency spectral embedding + k-means)

  Evaluates NMI and ARI on planted-partition networks with evolving
